# v0.1.0

Release date: 2026-02-27

## Highlights

- Initial public release of AWS Dashboard.
- One-command local deployment using Docker Compose (no repo clone required).

## Features

- Service coverage: EC2, EKS, Load Balancers (ALB/NLB), RDS, ElastiCache, OpenSearch, Amazon MQ, SES, Secrets Manager, IAM, and Cost Explorer (MTD totals, daily spend, and service breakdowns).
- Employee access workflow: employees request time-bound access to specific services.
- Manager approval: approve or deny requests with email tokens and OTP verification.
- JIT access: AWS STS credentials are issued per approved request and expire automatically.
- React frontend + FastAPI backend, backed by Postgres and Valkey for sessions and request history.

## Self-deployment (Docker Compose, no repo clone)

1. Create a new folder and enter it.
2. Create a `docker-compose.yml` with the contents below.
3. Create a `.env` file in the same folder with the variables listed after the compose file.
4. Run `docker compose up -d`.
5. Open the UI at `http://localhost:4000`.

```yaml
services:
  backend:
    image: mguptahub/cloud-dashboard-backend:v0.1.0
    env_file:
      - .env
    depends_on:
      - db
      - valkey

  frontend:
    image: mguptahub/cloud-dashboard-frontend:v0.1.0
    ports:
      - "4000:80"
    depends_on:
      - backend

  db:
    image: postgres:16
    environment:
      - POSTGRES_DB=${POSTGRES_DB:-aws_dashboard}
      - POSTGRES_USER=${POSTGRES_USER:-aws_dashboard}
      - POSTGRES_PASSWORD=${POSTGRES_PASSWORD:-aws_dashboard}
    volumes:
      - db_data:/var/lib/postgresql/data

  valkey:
    image: valkey/valkey:7
    volumes:
      - valkey_data:/data

volumes:
  db_data:
  valkey_data:
```

Optional ports (only if you want host access):
- Postgres: `"5432:5432"`
- Valkey: `"6379:6379"`

Required `.env` variables:

- `POWER_AWS_ACCESS_KEY_ID`
- `POWER_AWS_SECRET_ACCESS_KEY`
- `POWER_AWS_REGION`
- `BASE_ROLE_ARN`
- `ADMIN_EMAIL`
- `SMTP_HOST`
- `SMTP_PORT`
- `SMTP_USER`
- `SMTP_PASSWORD`
- `SMTP_FROM`
- `POSTGRES_HOST`
- `POSTGRES_PORT`
- `POSTGRES_DB`
- `POSTGRES_USER`
- `POSTGRES_PASSWORD`
- `VALKEY_URL`
- `SECRET_KEY`

Minimal `.env` example (fill in your real values):

