# InfraWatch v0.1.0

Release date: 2026-03-06

## Initial Release

InfraWatch is a central hub for monitoring AWS infrastructure and managing just-in-time access to cloud resources. It gives teams real-time visibility into 11 AWS services — from EC2 instances and EKS clusters to cost breakdowns — while enforcing least-privilege access through a request-approve-expire workflow. No shared credentials, no permanent keys, no manual cleanup.

### Highlights

- **One dashboard for everything** — EC2, EKS, RDS, ElastiCache, OpenSearch, MQ, SES, Secrets, IAM, Load Balancers, and Cost Explorer in a single pane with live metrics
- **Just-in-time access** — employees request time-bound access, managers approve via email, temporary STS credentials auto-expire
- **Zero passwords** — authentication is entirely email OTP, from login to approval flows
- **Deploy anywhere** — Docker Compose for quick setup, Helm chart for Kubernetes, with support for in-cluster or external PostgreSQL and Valkey

## Features

### My Dashboard
- Personalized monitoring with draggable, collapsible panels and live metric widgets
- Auto-refreshing charts (2-min interval) with 1h / 6h / 24h / 72h time ranges
- Maximize any widget to full-screen for detailed inspection
- Clone, edit, and delete widgets; layout persists across sessions

### AWS Service Tabs
| Tab | Capabilities |
|-----|-------------|
| **EC2** | Instance states, types, AZs, CPU utilization, detailed metrics |
| **Load Balancers** | ALB/NLB/CLB health, traffic metrics, request counts |
| **EKS** | Cluster status, nodegroup health, API server and scheduler metrics |
| **Databases** | RDS & DocumentDB health, connections, storage, IOPS |
| **ElastiCache** | Replication groups, cache clusters, memory and hit rates |
| **OpenSearch** | Domain health, JVM pressure, search and indexing rates |
| **MQ** | ActiveMQ & RabbitMQ broker status, connections, queue depth |
| **SES** | Sending quotas, suppression list management, identity/DKIM config |
| **Secrets** | Secret metadata, age, rotation status, on-demand value loading |
| **IAM** | Users, groups, policies, MFA status, access key audit |
| **Cost** | Month-to-date totals, daily spend charts, service breakdowns |

### Just-In-Time Access
- Employees request time-bound access (up to 12h) to specific AWS services
- Managers approve/deny via email link with OTP verification
- Temporary AWS STS credentials issued per approved request, auto-expire
- Admin panel for user management, service allowlists, and request audit log

### Security
- Email OTP authentication throughout (no passwords)
- Cryptographically secure OTP generation
- Rate-limited OTP endpoints with brute-force lockout
- Scoped IAM session policies per request (read-only, least-privilege)
- Session cookies with configurable Secure flag
- HTML-escaped email templates

### Infrastructure
- **Backend**: FastAPI, boto3, Celery (worker + beat) for background metric collection
- **Frontend**: React 18, Recharts, @dnd-kit for drag-and-drop
- **Database**: PostgreSQL for users, requests, and collected metrics
- **Cache**: Valkey (Redis-compatible) for sessions and API cache
- **Deployment**: Docker Compose (dev and production), Helm chart for Kubernetes
- Multi-region support (30 AWS regions)

## Deployment

### Docker Compose

```yaml
services:
  backend:
    image: mguptahub/infrawatch-backend:v0.1.0
    env_file: .env
    depends_on: [db, valkey]

  frontend:
    image: mguptahub/infrawatch-frontend:v0.1.0
    ports: ["4000:80"]
    depends_on: [backend]

  db:
    image: postgres:16
    environment:
      POSTGRES_DB: ${POSTGRES_DB:-awsdashboard}
      POSTGRES_USER: ${POSTGRES_USER:-awsdashboard}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD:?required}
    volumes: [db_data:/var/lib/postgresql/data]

  valkey:
    image: valkey/valkey:8-alpine
    command: ["valkey-server", "--requirepass", "${VALKEY_PASSWORD:?required}"]

volumes:
  db_data:
```

### Helm

```bash
helm repo add infrawatch https://infrawatchhelm.mguptahub.com/helm
helm install infrawatch infrawatch/infrawatch \
  --set aws.powerAccessKeyId=... \
  --set aws.powerSecretAccessKey=... \
  --set aws.baseRoleArn=... \
  --set app.adminEmail=admin@company.com \
  --set postgresql.auth.password=secret \
  --set valkey.password=secret
```

See [values.yaml](https://github.com/mguptahub/infrawatch/blob/main/helm/values.yaml) for full configuration options including external PostgreSQL and Valkey support.
